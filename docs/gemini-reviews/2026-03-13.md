# Code Review Report: Fictotum Staging (2026-03-13)

**Date:** Friday, March 13, 2026  
**Status:** **FAILURE (AWAITING SECURITY HARDENING)**  
**Scope:** Reviewing `staging` branch against `main` (Open & Social M1-M4 Push).

## Patterns Observed
1. **Broken Auth Assumptions**: The "Open the Gates" push removed the password gate but failed to replace it with a robust role-based access control (RBAC) system. The `auth()` session check is used as a universal "is admin" proxy, which is false now that public OAuth is active.
2. **Infrastructure Bypass**: New API routes (specifically analytics) are bypassing established library patterns (`lib/neo4j.ts`) and creating redundant, unoptimized database connections.
3. **Styling Regression**: Significant move away from Tailwind CSS towards complex inline styles in new components (`SeriesPage`, `AddToCollectionButton`), violating project-wide conventions.
4. **Cypher Performance Debt**: Increasing use of subqueries and string-parsing logic in Cypher that will fail to scale as the entity count moves toward the 3K+ milestone.

## Findings (Sorted by Severity)

### CRITICAL: Public Access to Administrative Interface
- **File**: `web-app/app/admin/layout.tsx`
- **Line Range**: 11-13
- **Severity**: **CRITICAL**
- **Description**: The admin layout check `if (!session?.user?.email)` only verifies that a user is logged in. Since the site now allows public sign-in via Google and GitHub, any user on the internet can sign in and access the `/admin` dashboard.
- **Consequence**: Full access to administrative tools for any authenticated user.
- **Suggested Fix**: 
  1. Add a `role` property to the `User` node in Neo4j.
  2. Map this role into the Auth.js session in `lib/auth.ts`.
  3. Change the check to `if (session?.user?.role !== 'admin') { redirect('/'); }`.
  4. Alternatively, use a whitelist of admin emails in an environment variable.

### HIGH: Unprotected Administrative Write API
- **File**: `web-app/app/api/admin/regenerate-image/route.ts`
- **Line Range**: 120-123
- **Severity**: **HIGH**
- **Description**: Like the layout, this API route only checks for a valid session, not an admin role.
- **Consequence**: Any logged-in user can trigger expensive Gemini 2.5 Image generation and overwrite existing illustrations in the database and Cloudflare R2, leading to significant cost and data vandalism.
- **Suggested Fix**: Implement a strict admin role check or email whitelist before processing the request.

### HIGH: Severe Performance Bottleneck in Series Metadata
- **File**: `web-app/lib/db.ts`
- **Line Range**: 1665-1675
- **Severity**: **HIGH**
- **Description**: The `getSeriesMetadata` query uses `size([(fig)-[:APPEARS_IN]->() | 1])` inside a `collect(DISTINCT ...)` block.
- **Consequence**: This subquery executes for every character appearance in every work in the series. For a large series, this results in thousands of sub-traversals per page load, causing high database latency and CPU usage.
- **Suggested Fix**: Pre-calculate the `total_portrayal_count` and store it as a property on the `:HistoricalFigure` node, or calculate the counts in a separate, optimized query using a single `MATCH (f:HistoricalFigure)-[:APPEARS_IN]->() RETURN f.canonical_id, count(*)`.

### MEDIUM: Inefficient Analytics Query Implementation
- **File**: `web-app/app/api/analytics/stats/route.ts`
- **Line Range**: 85-95, 107-115
- **Severity**: **MEDIUM**
- **Description**: The analytics queries use Cypher string functions like `split()` to parse JSON stored in the `event_data` property.
- **Consequence**: This is extremely CPU-intensive and forces Neo4j to perform a full scan and string manipulation for every event node in the range. It will not scale as the analytics dataset grows.
- **Suggested Fix**: Store `AnalyticsEvent` data as a proper property map in Neo4j or use `apoc.convert.fromJsonMap()` for more efficient parsing.

### MEDIUM: Database Connection Leakage in Analytics
- **File**: `web-app/app/api/analytics/stats/route.ts`
- **Line Range**: 46-50
- **Severity**: **MEDIUM**
- **Description**: The route manually creates a new Neo4j driver instance using `neo4j.driver(...)` instead of using the shared `getDriver()` from `@/lib/neo4j`.
- **Consequence**: Bypasses connection pool tuning (max 10 connections) and acquisition timeouts. Each request creates a new pool, which is extremely inefficient in a serverless environment and can lead to "Too Many Connections" errors on Neo4j Aura.
- **Suggested Fix**: Import and use `getSession` from `lib/neo4j.ts`.

### MEDIUM: Unauthenticated & Unprotected AI Discovery API
- **File**: `web-app/app/api/figure/[id]/discover/route.ts`
- **Line Range**: 12-100
- **Severity**: **MEDIUM**
- **Description**: The discovery agent route is entirely public and triggers a Claude Sonnet (`claude-sonnet-4-6`) request on every hit.
- **Consequence**: Vulnerable to bot spamming, which would rapidly deplete the Anthropic API budget. Note also that `claude-sonnet-4-6` is an invalid model name and likely results in an API error.
- **Suggested Fix**: 
  1. Correct the model name to `claude-3-5-sonnet-latest`.
  2. Implement rate limiting or require a session for this route.

### MEDIUM: Redundant Database Hits (Missing Cache)
- **File**: `web-app/app/series/[seriesId]/page.tsx`
- **Line Range**: 20, 39
- **Severity**: **MEDIUM**
- **Description**: `getSeriesMetadata` is called twice: once in `generateMetadata` and once in the page component.
- **Consequence**: Because `getSeriesMetadata` is not wrapped in `React.cache()`, it performs two identical, heavy database queries for every page request.
- **Suggested Fix**: Wrap the function in `React.cache` inside `lib/db.ts`.

### LOW: Violation of Styling Conventions
- **File**: `web-app/app/series/[seriesId]/page.tsx`, `web-app/components/AddToCollectionButton.tsx`
- **Line Range**: Various
- **Severity**: **LOW**
- **Description**: Extensive use of complex inline `style={{ ... }}` objects instead of Tailwind CSS utility classes.
- **Consequence**: Breaks the "Tailwind only" project mandate, makes themes harder to maintain, and increases the bundle size (styles are not shared).
- **Suggested Fix**: Refactor inline styles to Tailwind classes.
